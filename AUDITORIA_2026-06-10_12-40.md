## 1. Arquitectura

### Stack
```
Frontend:  React 19 + Vite 5 + Tailwind CSS + Framer Motion
Backend:   Node.js + Express + Sequelize + PostgreSQL
AI:        FastAPI + LangGraph + ChromaDB + Groq (LLaMA 3.3)
APIs ext:  OpenWeatherMap, NASA, AstronomyAPI
```

### Puntos Fuertes ✅
- Separación clara backend / frontend / AI service
- JWT con autenticación y roles en el payload
- CORS configurado con whitelist
- Rate limiting en auth (10/15min) y global (100/15min)
- Helmet con CSP, HSTS, etc.
- Proxy weather para no exponer API key en frontend

### Puntos Débiles 🔴
- **Dualidad de routing eliminada** ✅ (MainLayout/router.jsx borrados)
- **Sin capa de servicios**: controladores mezclan lógica de negocio con HTTP
- **SkyQualityZone**: ~50 campos planos, debería usar JSONB para datos semiestructurados
- **AI Service**: 1.1GB (entorno Python con ChromaDB local)
