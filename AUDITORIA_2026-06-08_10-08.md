## 1. Arquitectura

### 1.1 Estructura General ✅
```
Adastra Sky/
├── backend/          # Node + Express + Sequelize (REST API)
├── frontend/         # React 19 + Vite + Tailwind (SPA)
├── ai-service/       # FastAPI + LangGraph + ChromaDB
├── database/         # Seeds y documentos RAG
├── automations/      # n8n workflows
└── docs/             # Documentación
```

### 1.2 Hallazgos

#### 🔴 Problema Crítico — Sistema de Enrutamiento Dual
El frontend tiene **dos sistemas de layout incompatibles**:

- **App.jsx** (enrutamiento real): 14 rutas con componentes directos, sin `MainLayout`
- **MainLayout.jsx** (sin usar): Layout completo con header/nav/footer que renderiza `<Outlet />`
- **Router alternativo no utilizado**: `frontend/src/router.jsx` es un intento de React Router v7 con loaders/actions

**Impacto**: Si alguien activa `MainLayout`, las rutas de App.jsx dejarán de funcionar (sin `<Outlet />`). Si alguien activa `router.jsx`, todo App.jsx se vuelve obsoleto.

**Archivos**: `frontend/src/App.jsx`, `frontend/src/components/MainLayout.jsx`, `frontend/src/router.jsx`

#### 🟡 Medio — Backend sin Capa de Servicios
Las rutas (`backend/src/routes/`) llaman directamente a Sequelize y lógica de negocio. No existe una capa `services/` separada.

- `controllers/` y `routes/` tienen responsabilidades mezcladas
- Lógica de scoring duplicada (backend + frontend)
- Validaciones mezcladas con controladores

#### 🟡 Medio — SkyQualityZone con ~60 Campos
El modelo principal tiene demasiados campos planos en lugar de usar `JSONB` para datos semiestructurados (equipamiento, accesos, coordenadas). Cualquier extensión futura requiere migración.

#### 🟢 Info — AI Service bien aislado
FastAPI con routers separados, inyección de dependencias en `dependencies/`, RAG en `rag/`. Buena separación.

#### 🟢 Info — Variables de Entorno bien gestionadas
`backend/.env.example`, `ai-service/.env.example`, `frontend/.env.example` — todas documentadas.

### 3.2 Hallazgos

#### 🟡 Medio — Sin Migraciones
El backend usa `sequelize.sync({alter: true})` en lugar de migraciones formales. Esto significa:
- Cambios destructivos pueden ocurrir automáticamente
- No hay historial de cambios en la base de datos
- Rollbacks imposibles sin backup manual

**Archivo**: `backend/server.js` (sync en startup)

#### 🟡 Medio — SkyQualityZone.hemisphere sin Índice
Se filtra frecuentemente por hemisferio (`WHERE hemisphere = 'North'`) pero no tiene índice.

#### 🟡 Medio — Relaciones No Indexadas
`SkyQualityZone.islandId` y `Event.source` se usan en joins/filtros pero no tienen índices explícitos.

#### 🟢 Info — Experiencias bien indexadas
`Experience.userId` tiene índice FK, y `zoneId` también.

#### 🟢 Info — Seed con prompt interactivo
`seedUsers.js` pregunta antes de sobrescribir. Usa `bcrypt.hash` con saltRounds configurables.

### 3.3 Recomendaciones

1. **Añadir migraciones** con `sequelize-cli` o un sistema similar
2. **Índices compuestos**: `(islandId, category)` para SkyQualityZone
3. **JSONB** para campos como `equipment`, `access`, `coordinates` en SkyQualityZone
4. **Soft delete** en Experience (por si un usuario quiere "archivar" una experiencia)

#### 🟡 Medio — Sin CSRF Protection
Al ser SPA + API REST, el riesgo CSRF es bajo pero existe si algún endpoint acepta cookies de sesión. Actualmente no se usan cookies, solo JWT en header, así que riesgo bajo.

#### 🟡 Medio — requireAdmin.js: DB query en cada request
```javascript
// backend/src/middleware/requireAdmin.js
const user = await User.findByPk(req.userId);  // ← DB query cada vez
```
Mejor: poner `role` en el JWT payload y verificar sin DB query.

#### 🟢 Info — No hay refresh tokens
El JWT expira en 7 días. No hay mecanismo de refresh. Si se roba un token, es válido 7 días. Recomendación: añadir refresh tokens con rotación.

#### 🟢 Info — No hay password strength meter
Solo se valida longitud (mín 6 chars por Joi). Sin validación de mayúsculas/números/símbolos.

## 5. Rendimiento

### 5.1 Backend

#### 🟡 Medio — Sin Caché
- No hay Redis/Memcached
- `GET /api/sky/zones` devuelve todas las zonas (~100 registros) sin paginación
- `GET /api/chat/history` devuelve todo el historial sin paginación
- Las consultas a OpenWeather (proxy `/api/weather/current`) no tienen TTL

#### 🟡 Medio — Static Assets sin Caché HTTP
`/uploads/*` (imágenes de experiencias) se sirven sin `Cache-Control` headers.

#### 🟢 Info — Experiencias paginadas
`GET /api/experiences` tiene `limit` (default 50) y `offset`. Bien.

### 5.2 Frontend

#### 🟡 Medio — Bundle Grande
Reporte de build muestra:
- `index-*.js`: ~685KB total
- `chunk-vendors`: principalmente React, Leaflet, Framer Motion

#### 🟢 Info — Lazy Loading No Implementado
Todas las rutas se cargan eager. Con 14 rutas, convendría `React.lazy()` + `Suspense`, pero para un proyecto de este tamaño no es crítico.

#### 🟢 Info — Sin Memorización
Componentes como `SanctuaryPanel.jsx` y `ExperienceCard.jsx` se re-renderizan en cada cambio de estado del mapa. Considerar `React.memo`.

### 5.3 AI Service

#### 🟢 Info — ChromaDB local
La base vectorial es local (archivos en `rag/chroma_db/`). En producción con múltiples instancias, ChromaDB local no escala. Considerar ChromaDB Cloud o PGVector.

#### 🟢 Info — Sin rate limiting en endpoints AI
`/api/chat` y `/api/sky-score` no tienen límite de peticiones. Un usuario podría saturar la API de Groq.

## 10. Errores No Automatizables

### 10.1 Diferencia de Puerto (CORS)
```javascript
// backend/.env
FRONTEND_URL=http://localhost:5173
```
El usuario ejecuta en `http://localhost:3000`. Si no se actualiza `FRONTEND_URL`, el CORS bloqueará todas las peticiones del frontend. **Requiere acción manual del usuario**.

**Solución**:
```
FRONTEND_URL=http://localhost:3000
```

### 10.2 JWT_SECRET en Producción
```yaml
# render.yaml
- key: JWT_SECRET
  sync: false  # set manually in dashboard
```
El `JWT_SECRET` debe configurarse manualmente en el dashboard de Render. No hay fallback seguro si se olvida. La app arrancará pero los tokens serán impredecibles.

**Solución**: Añadir validación en `server.js` que lance error si `JWT_SECRET` no está definido en producción.

### 10.3 API Keys de Terceros
Tres servicios requieren API keys que el usuario debe obtener en sus dashboards:

| Servicio | Variable | Estado |
|---|---|---|
| OpenWeatherMap | `OPENWEATHER_API_KEY` | 🔴 Expirada o por rotar |
| NASA API | `NASA_API_KEY` | 🔴 No configurada |
| Groq (LLaMA) | `GROQ_API_KEY` | 🟡 Comentada para rotación |

**Impacto**: Sin OpenWeather, el proxy `/api/weather/current` falla. Sin Groq, el asistente IA opera en modo offline (solo RAG). Sin NASA, los eventos astronómicos no se sincronizan.

### 10.4 AI Service — ChromaDB en Producción
`CHROMA_PERSIST_DIR=./rag/chroma_db` usa almacenamiento local. En Render, los archivos se pierden en cada deploy. La base vectorial se reconstruye desde cero cada vez.

**Solución**: Migrar a ChromaDB Cloud o usar un volumen persistente en Render.

### 10.5 Astro Object `name` Duplicado
```javascript
// frontend/src/data/astronomicalData.js
const constellations = [
  { name: "Andrómeda", ... },
  { name: "Andrómeda", ... },  // ← duplicado
];
```
Dos objetos con exactamente el mismo nombre `"Andrómeda"` bajo la misma categoría. Causa confusión en la UI y en búsquedas.

**Solución**: Renombrar uno o eliminar el duplicado (verificar si hay diferencias en coordenadas/descripción).

### 10.6 Chat — Sin Validación de session_id
```javascript
// backend/src/routes/chat.js
const { message, session_id } = req.body;
// session_id se usa directamente en consultas sin validar formato UUID
```
Si un cliente envía `session_id: "'; DROP TABLE chat_histories; --"`, podría causar inyección SQL (aunque Sequelize escapa parámetros, es mala práctica).

**Solución**: Validar que `session_id` sea UUID válido con `uuid.validate()`.

### 10.7 Frontend — Sin Polyfills para Safari < 15
Algunas características usadas (como `getUserMedia()`, CSS `dvh`, `backdrop-filter`) no funcionan en Safari < 15 (iOS < 15). El target de `vite.config.js` no especifica navegadores antiguos.

**Solución**: Añadir `ios_saf >= 15` en `browserslist` o añadir polyfills condicionales.

### 10.8 Admin — No Verifica Admin Real en Registro
```javascript
// backend/src/routes/auth.js — POST /register
```

### 10.9 Experiences — Sin Límite de Subida
El multer accepta imágenes, pero no hay límite en:

### 10.10 Migración a Render

1. Crear el repo en GitHub
2. Actualizar la URL en render.yaml
3. Configurar manualmente JWT_SECRET y API keys en dashboard de Render
4. Configurar el frontend en Vercel con `VITE_API_URL` apuntando a Render
